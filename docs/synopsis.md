# Project Synopsis: Quantifying and Reducing Semantic Drift in Text-to-SQL Systems

## A Constraint-Driven Multi-Agent Approach

### Paragraph 1: Project Overview
In today's data-driven business environment, natural language interfaces to databases (NLIDB) promise to democratize data access by allowing non-technical users to query complex datasets using everyday language. However, existing text-to-SQL systems suffer from semantic drift—a phenomenon where natural language queries are misinterpreted, leading to inaccurate SQL generation, compliance violations, and unreliable results. This project addresses this critical problem by developing a constraint-driven multi-agent system that quantifies and minimizes semantic drift through rigorous validation and iterative refinement. Our proposed solution employs a five-agent pipeline orchestrated by LangGraph, integrating business ontology mapping, constraint validation, and a novel drift metric to ensure query accuracy and governance compliance. Leveraging technologies such as Python, FastAPI, SQLAlchemy, and NetworkX, the system transforms natural language queries into verified, executable retrieval plans while maintaining full provenance. This approach aligns with Sustainable Development Goal 9 (Industry, Innovation, and Infrastructure) by advancing data infrastructure that supports informed decision-making and operational efficiency in enterprises.

### Paragraph 2: Novelty / Innovation
What distinguishes this project from existing NLIDB solutions is its innovative multi-agent architecture with a critic loop for continuous semantic drift reduction. Unlike one-shot SQL generation models that lack feedback mechanisms, our system features a composite semantic drift metric combining intent alignment (40%), constraint adherence (30%), and result plausibility (30%), providing a quantitative measure of query fidelity. The critic loop iteratively refines agent outputs until drift converges below a threshold, enabling adaptive retrieval logic. Additionally, the integration of a formal business ontology represented as a NetworkX property graph grounds natural language elements to domain-specific schemas, resolving ambiguities through semantic similarity matching. This constraint-driven approach, coupled with embedding-based validation and anomaly detection, adds significant value by ensuring outputs are not only syntactically correct but semantically aligned with business rules and expectations, surpassing traditional transformer-based NL-to-SQL models in reliability and explainability.

### Paragraph 3: Societal / Practical Contribution
This project delivers substantial benefits to industry and society by enabling secure, self-service analytics without compromising governance or requiring SQL expertise. For enterprises, it reduces compliance risks, audit overhead, and data access barriers, fostering a culture of data-driven decision-making among business analysts. The system's rigorous drift detection and provenance tracking support regulatory requirements in sectors like finance and healthcare, where data integrity is paramount. Societally, by democratizing access to complex datasets, the project contributes to SDG 9 (Industry, Innovation, and Infrastructure) and SDG 12 (Responsible Consumption and Production) through more efficient resource allocation and informed policy development. Expected deliverables include a production-ready web API, interactive dashboards via Streamlit, and a containerized prototype deployable via Docker, demonstrating practical applicability. Following an Agile development methodology with comprehensive testing (pytest, 50+ unit tests), the project provides a scalable framework for future NLIDB advancements, ultimately enhancing data literacy and operational transparency across industries.</content>
